The "Role is required" error appearing even when a role is selected has been resolved.

Problem: React Hook Form wasn't properly recognizing the selected role value in the dropdown.
Solution: Enhanced the onChange handler to manually clear errors and trigger validation.

üîß What Was Fixed

Before (Problematic Code):

After (Fixed Code):
<select {...register('role')} onChange={(e) => { const value = e.target.value; setSelectedRole(value); // Manually trigger validation if (value) { clearErrors('role'); } else { setError('role', { type: 'required', message: 'Role is required' }); } trigger('role'); }} >
